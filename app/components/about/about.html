About

The joint military symbology explorer is an experiment created primarily for

  • learning the Joint Military Symbology Markup Language (JMSML)
  • experimenting with Javascript frameworks and tools
The JMSML project is an initiative by Esri, and provides a data encapsulation of MIL-STD-2525D and APP-6(C).

If you find bugs or have ideas for improving the symbology explorer, feel free to contact me or submit an issue on the GitHub project page.

Technical details

All of the symbol definitions and symbol SVG files are from the JMSML project. I have just converted the data to a Javascript-friendly format and assembled the different SVG files into symbols.

Please note that the symbology explorer is an experiment. I don't recommend using the techniques I have used in production for the following reasons:

  • Almost all of the JMSML data is loaded into memory (see jmsml.js) when the page is loaded. Unless you are writing a symbology explorer this approach is a waste of memory.
  • Drawing a symbol requires up to six different SVGs (and many server request). In a production system combine the SVGs on the server and send only one SVG file to the client.

Some tools and frameworks I have used for this project:

  • Python (2.7) for extracting information from the JMSML instance files and converting it to javascript
  • AngularJs for the web application
  • Bootstrap for styling (I'm sorry for using the default styles )
  • Gulp for building the project

Related projects

JMSML is not the only open source symbology project. If you need a symbol renderer check out one of these projects:

  • mil-sym-js MIL-STD-2525 symbol rendering JavaScript library for modern web applications (font based, 2525B, 2525C)
  • mil-sym-js MIL-STD-2525 B and C rendering library and web service
  • MilSymbol Military Unit Symbols in JavaScript (SVG based, APP6B, 2525C)
  • Ortelium A tool for generating military map symbols (Java, 2525B)
  • MilSym Provides C# WPF & Silverlight 4.0, XAML-based, Bing & Esri map symbology (2525C)
